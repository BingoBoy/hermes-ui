# 03 Teknisk arkitektur

# Formål

Denne siden beskriver teknisk arkitektur for Hermes UI for Bob.

# Anbefalt arkitektur

```
Klient
MacBook / iPhone / iPad

        HTTPS
          |
          v
Cloudflare Access

        Tunnel
          |
          v
Bob / Mac mini

        localhost
          |
          v
Hermes UI backend

        approved commands
          |
          v
Hermes / launchctl / logs
```

# Komponenter

## Frontend

Mulige valg:

- Enkel HTML først
- Next.js hvis UI-et skal vokse raskt

Anbefaling for første versjon:

- Start enkelt
- Unngå tung frontend før backend og sikkerhetsmodell er avklart

## Backend

Mulige valg:

- Flask
- FastAPI

Anbefaling:

- FastAPI hvis det skal bygges som API-first
- Flask hvis det skal være mest mulig enkelt og lite

# Backend-ansvar

Backend skal:

- Eksponere status-endepunkter
- Kjøre kun godkjente kommandoer
- Lese logger
- Sanitise output
- Ikke eksponere secrets
- Ikke tilby fri shell

# Foreslåtte API-endepunkter

```
GET  /api/status
GET  /api/hermes/status
POST /api/hermes/start
POST /api/hermes/stop
POST /api/hermes/restart
GET  /api/logs/hermes?lines=100
GET  /api/system
```

# Cloudflare

Cloudflare bør håndtere:

- Ekstern HTTPS
- Access-innlogging
- Tunnel til lokal port på Bob

Backend bør bare lytte lokalt:

```
127.0.0.1:8787
```

# Sikkerhet

- Ikke bind backend direkte til offentlig interface
- Bruk Cloudflare Access foran løsningen
- Bruk allowlist av kommandoer
- Loggfør handlinger
- Skjul tokens, passord og private nøkler fra loggoutput
- Ikke implementer fri terminal i MVP

# Deployment

Første versjon kan kjøres som:

- launchctl-tjeneste på macOS
- eventuelt Docker senere
