# 02 UI-wireframes

# Formål

Denne siden beskriver første skisse til brukergrensesnitt for Hermes UI for Bob.

# Prinsipp

UI-et skal være enkelt, driftspreget og praktisk. Første versjon bør ligne mer på et kontrollpanel enn en chat-app.

# Hovedvisning

```
+------------------------------------------------+
| Hermes UI for Bob                              |
| Status: Online                                 |
+------------------------------------------------+
| Bob-status                                     |
| - Hostname                                     |
| - Uptime                                       |
| - Disk                                         |
| - Minne                                        |
+------------------------------------------------+
| Hermes-status                                  |
| - Kjører / stoppet                             |
| - Sist restartet                               |
| - Aktiv prosess-ID                             |
+------------------------------------------------+
| Handlinger                                     |
| [Start Hermes] [Restart Hermes] [Stop Hermes]  |
+------------------------------------------------+
| Logger                                         |
| siste 100 linjer                               |
+------------------------------------------------+
```

# Navigasjon

Foreslått meny:

- Dashboard
- Hermes
- Logger
- Server
- Innstillinger

# Dashboard

Dashboardet skal gi rask status:

- Bob online/offline
- Hermes running/stopped
- Cloudflare tunnel-status
- Siste logglinjer
- Eventuelle varsler

# Hermes-side

Hermes-siden skal vise:

- Status
- Start/stopp/restart
- Kommandohistorikk
- Godkjente workflows

# Logger-side

Logger-siden skal vise:

- Hermes-logg
- Backend-logg
- Cloudflare-relatert status hvis tilgjengelig
- Filter for siste 50/100/500 linjer

# Server-side

Server-siden skal vise:

- Hostname
- Uptime
- Diskbruk
- Minnebruk
- CPU-belastning
- Relevante launchctl-tjenester
- Relevante Docker-containere hvis aktuelt

# Designretning

- Rent og rolig grensesnitt
- Store tydelige statuskort
- Få knapper
- Tydelig advarsel på handlinger som stopper tjenester
- Mobilvennlig layout, men desktop først
