# 01 Kravspesifikasjon MVP

# Formål

Denne siden beskriver første versjon av Hermes UI for Bob.

Målet med MVP-en er å lage et trygt og enkelt grafisk grensesnitt for Hermes/Bob, uten å åpne for fri terminaltilgang i nettleseren.

# MVP-mål

Første versjon skal gjøre det mulig å:

- Se om Bob er tilgjengelig
- Se om Hermes kjører
- Starte, stoppe eller restarte Hermes via godkjente handlinger
- Vise relevante logger
- Vise enkel systemstatus
- Bruke løsningen via Cloudflare Access

# Ikke med i MVP

Dette skal ikke være med i første versjon:

- Fri terminal i nettleseren
- Full agent-orchestrering
- Avansert brukerhåndtering utover Cloudflare Access
- Full historikkdatabase
- Mobiloptimalisert app utover responsivt webgrensesnitt

# Brukerhistorier

## Som Truls vil jeg se status på Bob

Slik at jeg raskt kan vite om serveren er oppe og tilgjengelig.

## Som Truls vil jeg se status på Hermes

Slik at jeg vet om assistent-/agentlaget kjører som forventet.

## Som Truls vil jeg kunne restarte Hermes

Slik at jeg kan løse enkle driftsproblemer uten SSH-terminal.

## Som Truls vil jeg kunne se logger

Slik at jeg kan forstå hva som skjer når noe feiler.

## Som Truls vil jeg bruke dette fra hvor som helst

Slik at Bob kan brukes via Cloudflare uten at jeg må være på lokalnettet.

# Akseptansekriterier

- UI-et er tilgjengelig lokalt på Bob
- UI-et er tilgjengelig via Cloudflare Access
- Status-endepunkt returnerer korrekt status
- Restart-knapp fungerer for Hermes
- Loggvisning fungerer uten å eksponere sensitive nøkler
- Ingen fri shell-kjøring er tilgjengelig i MVP
- Git-repo er opprettet og første versjon er committet

# Foreslåtte første handlinger

1. Kartlegg nøyaktig hvordan Hermes startes, stoppes og restartes i dag.
2. Lag en liste over trygge kommandoer som UI-et kan få lov til å kjøre.
3. Velg Flask eller FastAPI for backend.
4. Bygg status-endepunkt først.
5. Legg på enkel HTML/Next.js-visning etterpå.
